Eat with "else"

The rule I designed is around key pressing. The original "if" statement is that, once the max steps are reached, you can reset the sketch by pressing the space bar. I added an "else" statement where, if any key other than the space bar is pressed, an 'error' message appears instead of resetting the sketch.

The "otherwise" that happens is that a sassy error message shows up and the filled up canvas changes colours. These visual cues are also how the user knows the state has changed, albeit temporarily and only for as long as a wrong key is pressed.
