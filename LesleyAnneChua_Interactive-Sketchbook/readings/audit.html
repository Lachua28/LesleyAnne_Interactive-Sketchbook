Interface Audit — Micro-Project (Code Only)

Visibility of System Status

Does the page communicate where the user is?: Yes, with a heading.

Is it clear what stage or state they are in?: Yes, the bus game has text that says which level you are on.

Is it obvious what is interactive?: Mostly; the game has text that tells you whenever you can click or press a key to affect the system, but it does not specify if you need to click in a certain area (main weakness).

Affordances & Signifiers

What actions are available on this page?: Clicking and pressing any key, which are done to play the game.

How are those actions signified?: Text instructions within the P5 canvas.

Is anything clickable but unclear?: No, since clicking is not isolated to one area within the game and only has one function.

Is anything unclear but interactive?: No, since there are only two intended interactions, which are both instructed within the game.

Navigation & Mapping

How does a user move through this system?: Clicking with the mouse and pressing any key on the keyboard when required.

Does layout reflect system structure?: I’m not sure if this is applicable to my game. However, the layout does resemble what the game is supposed to be mimicking.

Is hierarchy clear?: Somewhat; the title and other important text are bigger to show hierarchy, but there’s not much hierarchy differentiators other than that.

Does interaction feel predictable?: Yes, but only because it is so limited.

Where might a user hesitate?: Users may be confused on whether they need to click in a certain area, or whether to click or use a key press at times.

Constraints & Assumptions

What does the page assume the user understands?: That the green area marked on the ‘road’ is the full area of the bus stop (rather than needing to align the bus to the middle of the bus sign).

What is hidden?: The overshoot/extra allowance of the bus stop “win” area; this means that you don’t need to fit the entire bus in the green area in order to win.

What requires guessing?: Where exactly you need to stop the bus (when first playing).

What is restricted?: I guess the win screen is restricted since you need to interact with the game a certain way to get to it?

Identify at least one assumption: Users might assume that clicking is the only interaction they need to do because of how simple the game is.

Heuristic Violations

Identify two Nielsen heuristics your page violates:

1. Heuristic #4: Consistency and Standards

Describe: A peer brought up during critique that it felt unintuitive to have the “continue” function of my game be triggered by a key press versus the click used to stop the bus. I coded the key press so you wouldn’t accidentally fail the next level if you accidentally clicked too much, but I think this technically counts as violating what users would typically expect from a click-based game.

Where it occurs: Whenever the bus is stopped (after a click).

Structural fix: Make it possible to continue with a click and/or key press.

2. Heuristic #10: Help and Documentation

Describe: The “wrong” red bus stop that appears after the first level appears with no context. It was designed with the assumption that you would know not to stop there because it doesn’t match the previous “winning” bus stop. However, some users may not understand that the red stop does not count as a winning stop. (Additionally, I realize these colours may not work for people who are colour-blind, so those users wouldn’t know why they are failing in the first place.)

Where it occurs: Levels 2–10 of the game.

Structural fix: Add additional text in a “Help” section to explain that you should avoid the red stops.

Structural Revision Proposal

Propose three structural changes to this page:

  1. State visibility: add more visual feedback for “fail” and “win” states.
  2. Navigation: Add site navigation to this page and/or a way to restart the page/game once you win the last level.
  3. Interaction Clarity: Use a different approach to explaining how to interact with the game, such as with imagery rather than text. Alternatively, use different text hierarchy to make the existing instructions easier to understand at a glance (e.g. “CLICK TO STOP BUS” rather than the full sentence I currently have.)
