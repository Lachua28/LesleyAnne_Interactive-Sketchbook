Interface System Statement (Draft)

Navigation model

The navigation model I decided to use on mobile is a separate screen that opens up after clicking a “hamburger menu” icon. The hamburger menu is visible on every screen within a sticky header bar at the top.

For the desktop screens, I decided to use drop-down menus. This groups and sections out the navigation more than mobile because there’s space for it in the header bar.

Hierarchy logic

For text, I primarily used size as my hierarchy differentiator. H1 is the largest heading while H4 is the smallest. Because H4 is the same size as the body text, it is bolded to show contrast. h3 and H3 are styled with “Semi-bold” and “Medium” weights respectively. Since they already differentiate from the size, the weight selections are for aesthetic purposes based on how I personally format my own writing (I like having H3 intentionally lighter than the other headings).

Mobile-first reasoning

Because of the amount of content on these three pages, it was important to start with mobile to streamline the information architecture. If the screen looked too cluttered, it meant that there was probably information there that was unnecessary to have. Starting with mobile meant that I could narrow down what was really necessary to include.

Then, when I created the desktop screens, I could theoretically expand more. However, in this case, I think the limited number of pages on my site (compared to, say, the Wall Street Journal) meant that I didn’t really need to add more elements anyway. The only change was the navigation model on desktop since there was more space in the header/nav bar for navigation links.

Norman

The connection my UI design has to Norman are signifiers. Norman provides an example of signifiers using an image of a screen with arrow symbols on a theoretical site. These arrows signify that you can swipe or move left and right between screens. There are also tabs on the top and bottom of the screen that have symbols to indicate menus.

In my design, I use a hamburger menu which is a universal symbol that you can open a menu on a website. I also chose to use directional arrows next to my dropdown menus to show that they can open. The arrows are something I took from the Canvas modules UI. Writing this makes me realize that the concept of arrows for dropdown menus also relates to Norman’s topic of mapping, since the arrow directions can indicate the direction the dropdown menu is in (e.g. down arrow means the menu has dropped down).

Nielsen

I believe my UI design matches Nielsen’s heuristics of “Visibility of System Status,” (intended) “User Control and Freedom,” and “Consistency and Standards.”

The “Visibility of System Status” is present in the menus on both mobile and desktop. The mobile menu makes it clear that you have a navigation menu open because it is both labelled and covers the entire screen with a background to cover whatever page you are currently on. On desktop, this is shown with the arrows on the dropdown menus changing to show that they are ‘open.’

“User Control and Freedom” is present in that you can access every page from every page because the navigation menus (hamburger and dropdown) are available on every page via the header/nav bar. This is only possible on desktop because I know there will be a set number of pages by the end of the semester.

Lastly, there is consistency in my text hierarchy. Although differing between the screen sizes, the mobile and desktop screens use the same text sizes and weights throughout every page. If I’m remembering correctly, the hamburger and dropdown menu symbols are also considered a “standard” on the web, so most people would understand what they mean.

Shneiderman

The Schneiderman rules that I think most applies to my project are 1) consistency (due to the reasons mentioned in the above section), and 2) reduce short-term memory load. I believe my project reduces short-term memory load, particularly in the desktop screens, because all the navigation is grouped on the nav bar on every screen. This, of course, doesn’t apply to mobile because it would be too much to fit on one screen. Other than the navigation, I think my website also includes all the information you need for each page, such as the description on the home page. The only downside is that it sacrifices minimalism somewhat, another heuristic/rule.
